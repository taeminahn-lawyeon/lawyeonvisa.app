# 새 글 작성 에디터 — 사용 안내

인사이트 글을 **처음부터** 로연 아티클 프로토콜 v2.1 형식으로 쓰기 위한 도구입니다.
주소: **`/blog-editor.html`** (관리자 대시보드 상단 **새 글 작성** 탭에서도 열림).

## 동작 방식 (중요)
이 에디터는 **서버에 저장하지 않습니다.** 글을 다 쓰면 **완성 · 출력**을 눌러
나오는 결과를 **Claude Code에 전달**하면, Claude Code가 기존 방식대로
`content/*.html` 저장 → `build-site.js` 등록 → 빌드 → `main` 배포까지 진행합니다.
(브라우저에 GitHub 토큰을 저장하지 않아 안전합니다.)

## 쓰는 순서
1. **언어**를 고릅니다(한국어/English/Tiếng Việt). 한 번에 한 언어씩 작성합니다.
2. **1~6번 카드**를 채웁니다.
   - 기본 정보: 분류, 읽는 시간, **슬러그**(영문 소문자·하이픈, 세 언어 공통), 상단 라벨.
   - 제목·요약, 도입(150~300자)·직답, 본문 블록(소제목 H2 3~5개 · 그중 하나는 경계 지점),
     자주 묻는 질문(1~2개), 소개 문단·고정 문안·이어지는 글.
   - 본문에서 **굵게**는 `**두 별표**`로 감쌉니다(프로토콜상 아껴 씀).
   - 오른쪽 **미리보기**가 실제 블로그 스타일로 즉시 렌더링됩니다.
3. **7번 자가 점검**의 항목이 모두 ✓인지 봅니다(소제목 개수·표 1개 이하·금지 표현 등).
4. **완성 · 출력** → **본문 HTML 복사** 또는 **HTML 파일 다운로드**, 그리고
   **등록 정보 복사**(발행월은 직접 지정)까지 함께 복사해 Claude Code에 붙여 넣습니다.
5. 다른 언어판이 필요하면 상단 **언어**를 바꿔 같은 슬러그로 다시 작성 → 출력.

## 브라우저를 닫아야 할 때
**임시 저장**을 누르면 이 기기(브라우저)에 초안이 보관되고, 다음에 **임시 불러오기**로
이어 쓸 수 있습니다. 다른 기기로는 옮겨지지 않습니다.

## 자동으로 채워지는 것(손대지 말 것)
목차(TOC) 앵커, 상담 콜아웃의 “상담 신청하기 →” 링크, 소개 문단·고정 문안의 기본값,
그리고 저장 이후 빌드가 넣는 BlogPosting·FAQPage JSON-LD, og:image, 언어 토글 등.

## 사진을 넣을 때
본문 **+ 사진** 블록의 경로는 `__BASE__images/blog/이름.jpg` 형식으로 적고,
**이미지 파일 자체는 Claude Code에 따로 전달**해야 합니다(에디터는 파일을 올리지 못함).
