# 블로그 시스템 — 디자이너 핸드오프

> **For**: 디자이너 에이전트
> **From**: 개발자 세션 (현행 시스템 관찰 기반)
> **Scope**: 사용자 블로그 목록/상세, 관리자 블로그 에디터, 정적 빌드 파이프라인
> **Not in scope**: 비즈니스 콘텐츠 전략, 카피라이팅, 마케팅 KPI

---

## 1. 시스템 개요 (30초 요약)

**하이브리드 아키텍처**: 동적(Supabase DB 기반) + 정적(GitHub Actions로 SSG).

```
[관리자] ─▶ admin-blog-edit.html ─▶ Supabase(blog_posts) ─▶ 두 갈래
                                                          ├─▶ 동적 조회: /blog.html?category=...  /blog-post.html?slug=...
                                                          └─▶ 매일 09:00 KST 빌드: /blog/{slug}.html (정적, SEO 유리)
```

**콘텐츠 모델**: 본문은 **9종 블록**으로 구성된 JSON 배열(`blog_posts.content`). 디자이너가 가장 주의할 부분은 **각 블록의 시각 스타일과 일관성**(섹션 6).

**현재 정적 파일 수**: 2개 (D-10 비자 가이드, F-2-R 지역특화 가이드).
**스타일 기조**: Toss 영향 (파랑 `#3182f6`, Noto Sans KR, 카드·라운드 코너 중심).

## 2. 페이지 · URL 맵

### 2-1. 사용자 페이지
| 경로 | 파일 | 역할 |
|---|---|---|
| `/blog.html` | `blog.html` | 목록 (전체) |
| `/blog.html?category={id}` | 동일 | 카테고리 필터 (URL은 JS 내부에서만 사용, 실제 카테고리 탭 클릭 시 URL 변경 없음) |
| `/blog-post.html?slug={slug}` | `blog-post.html` | 동적 상세 (정적 파일 있으면 자동 301 → `/blog/{slug}.html`) |
| `/blog/{slug}.html` | `blog/{slug}.html` | 정적 상세 (SEO 최적, 완전 자급 HTML) |

### 2-2. 관리자 페이지
| 경로 | 파일 | 역할 |
|---|---|---|
| `/admin-blog.html` | `admin-blog.html` | 목록 + 통계(전체·발행·임시·당월) + 검색·필터·삭제 |
| `/admin-blog-edit.html?id={id}` | `admin-blog-edit.html` | **블록 기반 에디터** (새 글/수정 공용) |
| `/admin-insert-blog.html` | `admin-insert-blog.html` | 일괄 삽입 유틸 (개발 목적, UI 디자인 우선순위 낮음) |

```

│  [로고] 법무법인 로연           [서비스] [뉴스&인사이트] │ ← sticky 헤더
```

### 3-2. 주요 상태 (디자이너 커버 필요)
- **로딩 중**: 스피너 (회전, 1s linear infinite)
- **빈 카테고리**: "해당 카테고리 글이 없습니다" 안내 일러스트·텍스트
- **오류**: 데이터 로드 실패 시 재시도 버튼 영역
- **썸네일 없음**: 카테고리 이모지 + 그라디언트 배경 폴백

### 3-3. 카드 스펙
- **사이즈**: 가변 (그리드에 의해 결정), 썸네일 고정 높이 180px
- **제목**: 2줄 말줄임 (`-webkit-line-clamp: 2`)
- **발췌**: 2줄 말줄임
- **클릭**: 카드 전체가 클릭 영역 (`onclick="location.href='blog-post.html?slug='"`)
- **호버**: `translateY(-4px)` + 섀도우 강화

## 6. 본문 블록 타입 9종 (디자이너 핵심 관심사)

각 블록은 JSON 객체, `type` 필드로 구분. 아래 각 블록의 **데이터 스키마 / 현재 DOM / 현재 스타일 / 재설계 포인트**를 정리.

### 6-1. `heading`
**스키마**: `{ "type": "heading", "level": 2, "content": "섹션 제목" }`
**현재 DOM**: `<h2 id="section-{n}">{content}</h2>` (현재 구현은 `level`을 무시하고 항상 h2 사용)
**스타일**: font-size 24px / 700 / 색 `#191f28` / 상단 여백 48px / 하단 16px / 목차 anchor 대상
**재설계 포인트**: h2/h3/h4 레벨 구분 시각화, 앵커 호버 아이콘.

### 6-2. `paragraph`
**스키마**: `{ "type": "paragraph", "content": "본문 텍스트 (HTML 허용)" }`
**DOM**: `<p>{content}</p>` (innerHTML로 주입 → `<strong>`·`<em>`·`<a>` 인라인 서식 허용)
**스타일**: 15-17px / 1.8 line-height / 색 `#4e5968` / 문단 간 여백 20px
**재설계 포인트**: 인용(`<blockquote>`)·각주 지원 여부 결정 필요.

### 6-3. `list`
**스키마**: `{ "type": "list", "ordered": false, "items": ["...", "..."] }`
**DOM**: `<ul>`/`<ol>` · 각 `<li>{item}</li>`
**스타일**: 기본 브라우저 스타일 + 왼쪽 padding 24px · 항목 간 6px
**재설계 포인트**: 체크리스트(✅) 변형, 중첩 리스트 지원 여부.

### 6-4. `numbered-section`
**스키마**: `{ "type": "numbered-section", "number": 1, "title": "소제목", "content": "본문" }`
**DOM**:
```html
<div class="block-numbered-section">
  <div class="number-badge">1</div>
  <div class="section-content">
    <h3 class="section-title">...</h3>
    <p class="section-text">...</p>
  </div>
</div>
```
**스타일**: flex 레이아웃 · 배경 `#f8f9fa` · 좌측 원형 뱃지(32px, 그라디언트 `#3182f6 → #1b64da`) · 본문 18/700 + 15/400
**용도**: "단계별 안내" (1단계, 2단계, ...) 시각 강조.
**재설계 포인트**: 모바일에서 뱃지·본문 수직 재배치 여부.

### 6-5. `table`
**스키마**: `{ "type": "table", "headers": ["열1","열2"], "rows": [[...], [...]] }`
**DOM**: `<div class="block-table"><table><thead>...<tbody>...</table></div>`
**스타일**:
- 컨테이너 `overflow-x: auto` (모바일 좌우 스크롤)
- `th`: 배경 그라디언트 `#1a1a2e → #16213e`, 텍스트 흰색, 14px/600
- `td`: 14px/400 · `#4e5968` · 하단 1px 선 `#e5e8eb`
- 짝수 행 배경 `#f8f9fa`
**재설계 포인트**: 큰 표의 모바일 카드형 대체 뷰, 첫 열 sticky 등.

### 6-6. `info-box`
**스키마**: `{ "type": "info-box", "style": "info|warning|danger|success", "content": "..." }`
**DOM**:
```html
<div class="block-info-box {style}">
  <span class="block-info-box-icon">ℹ️</span>
  <div class="block-info-box-content">{content}</div>
</div>
```
**스타일(4종)**:
| style | 배경 | 좌측 보더 | 아이콘 색 |
|---|---|---|---|
| info | `#e8f4fd` | `#3182f6` 4px | `#3182f6` |
| warning | `#fff8e6` | `#f59f00` | `#f59f00` |
| danger | `#ffeaea` | `#f03e3e` | `#f03e3e` |
| success | `#e6fcf5` | `#20c997` | `#20c997` |
**재설계 포인트**: 아이콘 일관성(이모지 vs SVG), 다크모드 고려 색 재정의.

### 6-7. `image`
**스키마**: `{ "type": "image", "url": "...", "alt": "...", "caption": "..." }`
**DOM**: `<figure class="block-image"><img src="{url}"/><figcaption>{caption}</figcaption></figure>`
**스타일**: 둥근 코너 12px · 섀도우 `0 4px 12px rgba(0,0,0,0.08)` · 캡션 배경 `#f8f9fa` · 색 `#6b7684`
**관찰**: 이미지 업로드 UI 없음 → 관리자가 외부 URL 수동 입력. 디자이너 재설계 우선 대상.
**재설계 포인트**: 드래그앤드롭 업로드, 자동 최적화(포맷·크기) 지시 여부.

### 6-8. `link-button`
**스키마**: `{ "type": "link-button", "url": "...", "text": "버튼 라벨" }`
**DOM**: `<a href="{url}" class="block-link-button" target="_blank">{text} →</a>`
**스타일**: 배경 `#3182f6` · 흰 글씨 · 패딩 12×24 · 라운드 8px · 600 weight · 호버 `#1b64da`
**용도**: 본문 중간의 CTA (외부 링크 안내·신청 유도 등).

### 6-9. (9번째) — 현재는 **8종 + 레거시 HTML 문자열** 공존
- 위 8종 외에 **레거시 HTML 문자열**로 저장된 글도 존재 (JSON.parse 실패 시 HTML로 간주해 그대로 삽입).
- 디자이너 입장에서는 **8종 표준 블록 기준으로 설계**하되, 레거시 HTML 폴백 케이스를 "본문 그대로 스타일" 범위로 처리.

### 10-1. 지원 범위

## 11. SEO · 메타 구조

정적 파일(`/blog/{slug}.html`)은 렌더 시 아래 메타를 자동 삽입(빌드 스크립트 기반):

- `<title>{title} | 법무법인 로연</title>`
- `<meta name="description" content="{meta_description 또는 excerpt}">`
- `<meta name="keywords" content="{카테고리별 비자 키워드 세트}">`
- **Open Graph**: `og:type=article`, `og:title`, `og:description`, `og:image` (featured_image), `og:url`
- **Twitter Card**: `summary_large_image`
- **Schema.org JSON-LD**: `Article` 타입 (headline, description, image, author=법무법인 로연, datePublished, dateModified)
- **Canonical**: `<link rel="canonical" href="/blog/{slug}.html">` (동적 페이지 → 정적 페이지 정규화)

### 디자이너 영향
- **OG 이미지 권장 비율**: 1200×630px.
- **썸네일 vs OG 이미지**: 현재는 동일 이미지 사용 가능, 분리 디자인 원할 시 `thumbnail_url`와 `featured_image` 각각 지정.

## 12. 빌드 · 배포 파이프라인

### 12-1. 파이프라인 개요
```
(수동/트리거) ──▶ GitHub Actions (.github/workflows/build-blog.yml)
                  ├─ 매일 09:00 KST cron
                  └─ scripts/build-blog.js 수정 시
                 │
                 ▼
        node scripts/build-blog.js
                 │
                 ├─ Supabase `blog_posts` 전체 조회
                 ├─ 각 글마다:
                 │    ├─ 블록 → HTML 변환(렌더러 동일 로직)
                 │    ├─ 메타 태그·Schema.org 삽입
                 │    └─ `/blog/{slug}.html` 파일 생성
                 ├─ `sitemap.xml` 갱신
                 └─ git commit & push → GitHub Pages 자동 배포
```

### 12-2. 정적 vs 동적 차이
- **정적(`/blog/{slug}.html`)**: 스타일을 `<style>` 태그로 **인라인화**해 자급. 크기 50-60KB.
- **동적(`/blog-post.html`)**: 외부 `css/*.css` 참조 + `js/*.js`에서 블록 렌더링.
- 두 경로가 **같은 블록 렌더 로직**을 가져야 시각 일관성 유지 → 디자이너가 블록 스타일 업데이트 시 **두 곳(또는 공통 CSS)**에 반영 필수.

### 12-3. 빌드 지연
글 수정 후 정적 파일 반영까지 최대 **24시간**(다음 cron 실행까지). 즉시 반영 필요 시 관리자가 수동으로 Actions 재실행.

## 13. 제약 사항 · 재설계 시 유의점

### 13-1. 기술 제약
- **프레임워크 없음**: 순수 HTML + Vanilla JS. React·Vue 컴포넌트 도입 시 전체 스택 변경 수반 → 디자인 범위에서는 **CSS·DOM 패턴 개선**에 집중.
- **블록 에디터 UX**: 현재 매우 단순한 add/remove 수준. WYSIWYG 개선 여지 큼.
- **이미지 업로드 UI 없음**: URL 직접 입력만 가능. Storage 버킷(`blog` 또는 전용)은 별도. 업로드 UI 신설 필요.
- **관리자 인증**: 클라이언트 측 sessionStorage 기반 간이 인증이 운영 중이라는 관찰 있음. 디자인 개선 시 보안 팀과 협의 권장(이 문서에서는 자격 정보 미공개).
- **빌드 매일 1회**: 즉시 반영이 필요한 경우 "수동 재빌드" 버튼 UX 제안 가치 있음.

### 13-2. 콘텐츠 제약
- **법률·출입국 관련 콘텐츠**: 과장·결과 보장 표현 금지(변호사광고규칙). 디자이너가 임의로 "성공률 98%" 같은 카피 삽입하지 말 것.
- **가격 정보 노출**: 일부 서비스는 가격 언급이 금지된 영역(사업이민 등). 블로그 내 CTA에서 서비스 연결 시 가격 노출 여부를 관리자에게 확인.
- **카테고리 7종 고정**: 신설/병합은 관리자와 협의 필요 (관련 JS·i18n·이모지 매핑 동반 수정).

### 13-3. 접근성·국제화
- 목차·모바일 토글에 **키보드 내비·포커스 링·ARIA 속성** 보강 여지.
- 언어 전환 시 레이아웃 변동 최소화(텍스트 길이 차이 대응).
- `Noto Sans KR` 외에 다국어(베트남어·중국어·일본어·몽골어·태국어) 렌더링 체크 필요.

### 13-4. 레거시 콘텐츠
- 블록 배열이 아닌 **레거시 HTML 문자열**로 저장된 글이 공존 가능. 이에 대한 "본문 그대로 스타일" CSS는 유지해야 깨지지 않음.
- URL 301 리다이렉션 맵(구 slug → 신 slug)이 상세 페이지 코드에 있음 — 디자이너 재설계 시 직접 건드리지 않아도 됨.

### 13-5. 디자이너가 **건드리지 말 것**
- `slug` 규칙·URL 구조(SEO 이관 리스크)
- Schema.org·OG 메타 필드명(검색 엔진 의존)
- 블록 JSON 스키마 필드명(데이터 호환성). 스타일·DOM 구조·클래스명은 자유 변경 OK.
